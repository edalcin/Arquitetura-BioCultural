# Progresso da Migração v3.1 — SQLite com JSON

> Fonte de verdade de progresso entre sessões da migração v3.1. Antes de continuar o trabalho, leia esta checklist.

Este documento espelha o rastreador de progresso do plano de migração (Arquitetura BioCultural v3.0 → v3.1:
substituição de MongoDB por SQLite com JSON em cada unidade federada). Cada etapa é marcada `[x]` ao final de sua
execução, na mesma sessão em que é commitada no `main`.

## Checklist

- [x] **Etapa 1** — Decisão perene (ADR-005) + documentação da arquitetura v3.1 (repo `Arquitetura-BioCultural`).
- [x] **Etapa 2** — Harmonização documental dos repos greenfield (`BioCultRelatos`, `pluriverso`).
- [x] **Etapa 3** — `BioCultDB`: migração de código MongoDB → SQLite+JSON.
- [x] **Etapa 4** — `BioCultDB`: migração dos dados de produção MongoDB → SQLite (28/28 registros migrados e verificados).
- [x] **Etapa 5** — `BioCultTermos`: migração de código MongoDB → SQLite+JSON (mesmo banco do BioCultDB).
- [x] **Etapa 6** — `BioCultPapers`: storage local → SQLite+JSON + exportação de arquivo; import no BioCultDB.
- [x] **Etapa 7** — Empacotamento por unidade federada: um container único por unidade, SQLite compartilhado. Unidade de Fontes Secundárias (BioCultDB + BioCultTermos submodule) implementada e testada (build + run reais: 5 portas respondendo, `biocultdb_records`/`etnotermos*` coexistindo no mesmo arquivo, `AcquisitionService` lendo dados do BioCultDB no mesmo banco). Unidade Comunidade documentada (mesmo padrão, materializa quando `BioCultRelatos` existir).
- [ ] **Etapa 8** — Verificação final + regeneração dos grafos graphify.

## Notas

- Dependências: Etapas 1 e 2 são independentes entre si (fazer primeiro). Etapa 3 precede 4, 5, 6 e 7. Etapa 5
  precede 7. Etapa 6 depende de 3 (formato de import). Ordem recomendada: 1 → 2 → 3 → 4 → 5 → 6 → 7 → 8.
- Detalhes completos de cada etapa (escopo, arquivos, critérios de verificação) estão no plano de execução da
  migração e no `docs/architecture-decisions/ADR-005-sqlite-json-persistence.md`.
- Ao concluir uma etapa: marcar `[x]` aqui, commitar no `main` (nunca criar branch), e seguir para a próxima.
